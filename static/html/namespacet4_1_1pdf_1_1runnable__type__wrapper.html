kCannotCancel 

kCannotCancel: All messages on the UI thread will fall under this category. Since the UI gestures will be responded to immediately, we will neither be able to nor want to cancel the messages on the UI thread. This is also the default category for all runnnables. So, runnables must override the getType method below if they want their messages to be cancelable. Some messages on the background thread may also fall in this category.
