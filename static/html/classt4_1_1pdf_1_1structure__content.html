Detailed Description

Represents a "content item" in a structure tree. See the PDF 2.0 spec (ISO 32000-2:2020(E)), section 14.7.5.1.1 "Content items."

Briefly, a structure tree is a tree of structure elements. A structure element contains 0 or more child structure elements and 0 or more content items. Each content item represents a single marked content sequence, identified by a Marked Content ID (MCID).

A given marked content sequence may contain multiple content elements: text, images, paths, etc.

This class provides access to the content elements associated with the single MCID for a content item.

Returns the content associated with this structure_content.

When text is encountered it can be extracted in one of two modes.

  • When extracting for "visual fidelity" tricks may be played with Unicode values so that the correct glyphs appear when displayed with a converted font. Expanded ligatures will be recomposed into a single Unicode value. Duplicate Unicode mappings will be replaced with unique values. This mode is used when extracting text to create HTML for Dynamic View.
  • When extracting for "content fidelity" the original Unicode values from the font dictionary are maintained.

The text extraction mode is determined by the contents of the optional 'converted_fonts' parameter. If the text to be extracted is in a font in this set then it is extracted for "visual fidelity." Otherwise it is extracted for "content fidelity."

Note that 'converted_fonts' has meaning only for text content (doing nothing for other types, such as images). It also has no meaning for text content unless the caller actually retrieves the text. Unless callers are actually planning to get text out of the structure_content they should just us the default of 'none'.
